# スライド作成ルール

このルールは `slides/` 配下のMarkdownファイルを編集する際に適用されます。

## 基本構造

すべてのスライドは以下のフロントマターから始まります。テーマファイルへのパスは相対パスで指定し、`paginate: true` でページ番号を表示します。

```markdown
---
marp: true
theme: ../../themes/3shake-theme.css
paginate: true
---

## タイトル

### サブタイトル

<div style="font-size: 0.75em;">

コンテンツ

</div>
```

## スライドの種類

**通常スライド** は `---` で区切り、タイトルとサブタイトル、本文で構成します。

```markdown
---

## タイトル

### サブタイトル

<div style="font-size: 0.75em;">
コンテンツ
</div>
```

**トランジションスライド** はセクション間の切り替えに使用します。暗い背景に白文字で、聴衆の注意を引きつけます。

```markdown
---

<!--
_backgroundColor: #0a1929
_color: white
_class: transition
-->

<div style="display: flex; justify-content: center; align-items: center; height: 100%; flex-direction: column; color: white;">

## タイトル

**サブテキスト**

</div>
```

**タイトル/終了スライド** はプレゼンテーションの開始と終了に使用します。ブランドロゴと背景画像を配置し、発表者情報を含めます。

```markdown
---

<!--
_backgroundColor: #0a1929
_color: white
_class: title dark
-->

![bg](../../assets/images/3shake-background-full.png)

<img src="../../assets/images/3shake-logo.png" alt="3-SHAKE logo" style="position: absolute !important; top: 100px !important; left: 100px !important; width: 240px !important;">

<div class="title" style="text-align: left; margin-top: 100px; margin-left: 20px;">

# タイトル

### サブタイトル

</div>

<div class="author-info">
2025/XX/XX イベント名</br>
@your_name 発表時間
</div>
```

## レイアウトパターン

**2カラムレイアウト** は比較や対比を示すのに効果的です。

```markdown
<div style="display: flex; gap: 40px;">
<div style="flex: 1;">

**左カラム**

説明文をここに記述します。

</div>
<div style="flex: 1;">

**右カラム**

対応する説明文をここに記述します。

</div>
</div>
```

**画像 + テキストレイアウト** は視覚的な説明を添えるのに適しています。

```markdown
<div style="display: flex; gap: 40px;">
<div style="width: 35%;">
<img src="path/to/image.png" alt="説明" style="width: 100%;">
</div>
<div style="flex: 1;">

メインコンテンツをここに記述します。

</div>
</div>
```

## ボックスの配色

シンプルなグレー系の階層構成を使用します。カラフルな背景色（ピンク、緑、オレンジ背景など）は避けてください。

| レベル | 背景色 | 用途 |
|-------|--------|------|
| 通常 | `#f5f5f5` | 通常のボックス |
| 強調 | `#e0e0e0` | 💡ヒントボックス |
| 最強 | `#e0e0e0` + オレンジ文字 | パンチライン |

通常のボックスには `#f5f5f5` を使用します。

```markdown
<div style="background-color: #f5f5f5; padding: 15px; border-radius: 8px;">
内容
</div>
```

ヒントや強調には `#e0e0e0` を使用します。

```markdown
<div style="background-color: #e0e0e0; padding: 10px; border-radius: 5px; text-align: center;">
💡 <strong>メッセージ</strong>
</div>
```

最も伝えたいパンチラインには、グレー背景にオレンジ文字を組み合わせます。

```markdown
<div style="background-color: #e0e0e0; padding: 15px; border-radius: 8px; text-align: center; font-size: 1.3em;">
<span style="color: #e65100; font-weight: bold;">最も伝えたいメッセージ</span>
</div>
```

## コンテンツが溢れる場合の対処

コンテンツがスライドに収まらない場合、まずスライドを分割することを検討してください。これが最も推奨される方法です。

分割が難しい場合は、フォントサイズを段階的に下げます。0.65em より小さくする必要がある場合は、内容を見直すか分割を再検討してください。

| サイズ | 用途 |
|-------|------|
| 0.8em | 余裕のあるスライド |
| 0.75em | 標準（推奨） |
| 0.7em | やや詰まったスライド |
| 0.65em | 限界（これ以上は分割推奨） |

padding や margin を 15px から 12px、10px と減らす方法もありますが、見栄えとのバランスを考慮してください。

| パターン | 対策 |
|---------|------|
| 3カラム + 説明文 + ヒント | 説明文を削除 or 分割 |
| 表が5行以上 | 3行程度に絞る or 分割 |
| 各カラム5項目以上 | 2〜3項目に絞る or 分割 |

## Marpの技術的制限

### CSSの制限事項

Marpでは `position: absolute` や `position: sticky` でのタイトル固定は困難です。Marpはスライドを `<section>` 要素としてレンダリングし、PDF出力時にCSSの一部が無視されます。`position: absolute` を使うとコンテンツが重なる問題も発生します。

タイトル固定は諦め、通常のドキュメントフローを使用してください。コンテンツが多い場合はスライドを分割するのが正解です。

### トランジションスライドのクラス指定

暗い背景のトランジションスライドには `_class: transition` を追加して、通常スライドと区別してください。

```markdown
<!--
_backgroundColor: #0a1929
_color: white
_class: transition
-->
```

### 画像の貼り方

画像はHTMLの img タグを使用してください。div 内で Markdown の画像記法を使うと表示されないことがあります。

```markdown
<div style="text-align: center;">
<img src="../../assets/images/2025/presentation-name/image.png" alt="説明" style="max-width: 80%;">
</div>
```

## プレゼンテーション構成の原則

### 全体構成

プレゼンテーションは、タイトル、自己紹介（必要に応じて）、目次、「この発表で解決できること」、本編、まとめ、終了スライドの順で構成します。「この発表で解決できること」は聴衆の期待を設定する重要なスライドです。

本編は複数のセクションで構成し、各セクション間には繋ぎスライドを入れて文脈を繋げます。

### 情報の配置順序

| 情報の種類 | 配置場所 | 理由 |
|-----------|---------|------|
| Why（なぜやるべきか） | 本編の最初 | 動機づけが先 |
| What（何をするか） | Whyの後 | 動機が固まってから具体論 |
| How（どうやるか） | Whatの後 | 何をするか決まってから方法論 |

### 繋ぎスライドの重要性

セクションが変わる際は繋ぎスライドを入れて文脈を繋げます。繋ぎスライドでは、前セクションの要約（「〜はわかった」）、次への問い（「では、〜は？」）、次に紹介する内容の予告を含めます。

| パターン | 繋ぎの例 |
|---------|---------|
| Why → What | 「価値はわかった。では、何を？」 |
| What → How | 「ネタは見つかった。では、どう？」 |
| How → Practice | 「書き方はわかった。あとは実践」 |

## AIっぽさを避ける

AIが生成したコンテンツには特有の癖があり、聴衆はそれを敏感に感じ取ります。以下のパターンは避けてください。

**箇条書きの多用はNG。** 3つの項目が均等に並んでいると、それだけでAI生成を疑われます。情報を文章で繋げるか、あえて2つか4つにするか、重要度に差をつけてください。

**「まず」「次に」「最後に」の機械的な接続はNG。** 人間は話すとき、こんなに整然と順序立てません。「ところで」「実は」「ただ」といった、思考の揺らぎを感じさせる接続詞を混ぜてください。

**すべての項目に同じ分量の説明を付けるのはNG。** 本当に伝えたいことには厚く、そうでないことには薄く。メリハリのなさはAIの特徴です。

**「〜することが重要です」「〜が求められます」の連発はNG。** 誰に対して言っているのか、なぜ重要なのかを具体的に。抽象的な当たり障りのない表現はAIっぽさの象徴です。

**完璧に整った構成はNG。** 人間のプレゼンには脱線があり、余談があり、「これは余談ですが」という前置きがあります。少しの崩れが人間味を生みます。

## 「引き算」の重要性

良いプレゼンテーションは「何を書くか」ではなく「何を書かないか」で決まります。1スライド1メッセージを守り、「あれもこれも」と詰め込むことは避けてください。焦点がぼやけます。

「これを削ったら成り立たなくなるか？」と自問し、答えが No なら削ってください。AIは情報を足すのが得意ですが、人間の仕事は引くことです。

## 読者との情報の非対称性

書き手と読者の間には知識の差があります。これを意識せずに書くと読者は置いてけぼりになります。

専門用語には説明を添え、「なぜそうするのか」を書き、前提条件を明示してください。読者が再現できる情報を書くことを心がけてください。

## ファクトチェック

プレゼンテーションで誤った情報を発信すると信頼性を損ないます。

| 項目 | 確認方法 |
|-----|---------|
| 数値・統計 | 一次ソースを確認、年度を明記 |
| 引用文 | 原典で確認 |
| 技術仕様 | 公式ドキュメント、バージョンを明記 |
| 人名・書籍名 | 正式名称を確認 |
